# ブラウザのみで動画エンコードを実現するWebアプリ企画書

## 1. タイトル・提案者・日付
- **プロジェクト名**：ブラウザのみで動画エンコードを実現するWebアプリ
- **提案者**：島村 光輝（スーパーITエンジニア スーパーIT科）
- **日付**：2025年12月9日

## 2. プロジェクトの背景と目的
### 背景（現状の問題点）
- 動画エンコードは従来、専用ソフトや高性能サーバーが必要であり、Webブラウザ単体での処理はほぼ不可能とされてきました。
- サーバー負荷や運用コストが高く、ユーザー側の操作性も複雑。
- 要件としてPHP/MySQL/MVCを利用する必要があるため、掲示板機能を組み込みつつ、動画エンコードをテーマに選定。

### 目的
- サーバーサイド処理を最小限にし、ブラウザのみで1080p動画のエンコードを実現。
- 掲示板機能を通じてエンコード設定の共有・比較を可能にし、コミュニティ性を高める。

## 3. ターゲットとニーズ
- **ターゲット**：
  - 動画編集初心者：簡単操作で高品質なエンコードをしたい。
  - 開発者：WebCodecs APIの技術検証を行いたい。
- **ニーズ**：
  - ソフトインストール不要で、ブラウザだけで動画エンコード。
  - エンコード設定を簡単に選択・共有できるUI。
  - コミュニティで最適な設定を探せる仕組み。

## 4. 提案内容（ソリューション）
- **ブラウザのみで動画エンコード**：WebCodecs APIを活用し、1080p動画をエンコード可能。
- **掲示板機能**：エンコード設定を共有・比較し、ユーザー間で最適化。
- **メリット**：
  - サーバー負荷軽減 → 運用コスト削減。
  - ユーザーは追加ソフト不要 → 利便性向上。
- **独自性**：
  - WebCodecs APIを実用レベルで活用する国内初の事例。
  - UI設計にAI支援を導入し、操作性を最適化。
