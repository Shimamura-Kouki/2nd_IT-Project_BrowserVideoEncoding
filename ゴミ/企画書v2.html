Server-less Video Optimizer
ローカルリソースの限界に挑む・究極のブラウザ動画ラボ

「サーバー不要・通信不要・インストール不要」
Webブラウザだけで、どこまでプロフェッショナルな動画処理が可能か？
その技術的限界を追求し、実証する実験的プラットフォーム。

1. 企画の背景・目的

すべての処理をクライアントサイド（ローカル）で行うことにこだわり、プライバシー保護とサーバーコストゼロを実現します。
特に今回は、ブラウザ標準AI「Gemini Nano」の実証実験を組み込み、外部モデルのダウンロードすら不要な「真のゼロ・レイテンシーAI」の可能性を探ります。

2. ターゲットユーザー

  • 開発者自身 (60%): 理想のエンコード環境と、最新Web技術の検証を兼ねて。
  • ギーク・技術愛好家: 「ChromeにAIが載ったらしいけど実際どうなの？」という疑問への回答を実機で体験したい層。
  • 一般ユーザー: 動画や画像を安全かつ高速に軽量化したい層。

3. 実装機能一覧 (Full Spec)

メディア処理

  • Dual Encode Engine: 爆速WebCodecs (Video/Audio) と、高互換ffmpeg.wasmの切り替え。
  • Image Optimizer: サムネイル・資料画像の圧縮機能。
  • Media Analysis: MediaInfo.jsによる詳細解析。

インテリジェンス (3-Tier AI)

  • Experimental: Gemini Nano (Built-in AI)
    Chrome標準搭載のAIモデルをAPI経由で制御。モデルDL不要、通信不要、究極のローカルAIを実証実験として実装。
  • Fallback AI: Nanoが使えない場合は、WebLLM (Local) または Gemini Cloud API へ自動で切り替えてサポートを継続。

安定性 & UI

  • Memory Manager: モバイル端末でのクラッシュを防ぐメモリ解放システム。
  • Geek Dashboard: 処理状況をリアルタイムグラフで可視化。

4. 技術スタック

  • Frontend: HTML5, WebCodecs, ffmpeg.wasm, PWA
  • AI Layer: window.ai (Gemini Nano), WebLLM, Gemini API
  • Infrastructure: Tailscale (Dev), Sakura Server (Prod)

5. ロードマップ

  • Phase 1: Tailscale環境構築、PWA基盤、WebCodecs実装。
  • Phase 2: Gemini Nano (window.ai) の動作検証とAIフォールバック回路の構築。
  • Phase 3: UI作り込み、Geekモード、Gist連携。
  • Phase 4: 実機ストレステストと品質向上。
