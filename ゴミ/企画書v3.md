
# Server-less Video Optimizer (v3.1)

**〜AIが自己修復する、サーバーレス動画変換の実験場〜**

## 1. 企画概要

本プロジェクトは、サーバーサイド処理を行わず、ブラウザのリソースのみで動画エンコード・解析を行うWebアプリケーションである。
※Gemini Nano等の最新技術については、実装前の技術検証（PoC）を行い、実用性が確認できた場合のみ採用する。

## 2. ターゲット & コンセプト

* **一般ユーザー:** 「詳しくないけど、AIにお任せで動画が軽くなった！」という魔法のような体験。
* **ギーク/開発者:** 「ブラウザだけでここまで動くのか」「AIが勝手にデバッグしてる」という技術的驚き。

## 3. コア体験 (UX)

基本は「シームレス展開型」。Geek Switch ONで裏側の機構（ダッシュボード・コンソール）を開放する。

### Phase 1: エントリー & 解析

* ファイル選択後、動画を解析。単純な変換はルールベースで即時実行。

### Phase 2: Intelligent Optimization (検証対象)

* **Adaptive AI:** 高性能な環境（Cloud/Local LLM）では、複雑なFFmpegコマンド生成に挑戦。
* **Self-Healing (自己修復):** エラー発生時、AIがログを読んで修正するプロセスを「実況」として見せる。

### Phase 3: The Souvenir (持ち帰れる体験)

* 生成された設定（Command）や、Python NotebookをLitterbox経由で共有し、QRコードで持ち帰る。

## 4. 技術スタック & 検証計画

* **Frontend:** HTML5, WebCodecs, ffmpeg.wasm
* **AI Layer (PoC対象):**
  * **Candidate 1: Gemini Nano (window.ai)** - Chrome標準AI。事前の動作実験で「失敗」と判断された場合は実装しない。
  * **Candidate 2:** WebLLM (Local WebGPU) - 代替ローカルAI。
  * **Candidate 3:** Gemini API (Cloud) - 高性能フォールバック。
* **Dev Infra:** Tailscale (HTTPS/MagicDNS)

## 5. ロードマップ (Go/No-Go判断)

1. **Phase 0: 技術検証 (PoC)** - Chrome Canary環境にて Gemini Nano の動作・精度確認。 **→ 採用可否の決定**
2. **Phase 1: 基盤開発** - UI、ffmpeg.wasm、Litterbox連携の実装。
3. **Phase 2: AI実装** - PoCを通過したモデルのみを組み込む。
