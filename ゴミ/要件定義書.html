要件定義書 v3.2 (Functional Restoration)

Version: 3.2
Scope: AI技術検証(PoC)と、実用的なWebアプリ機能の完全統合

1. 開発フェーズ定義 (Hybrid)

AI機能の不確実性（実験要素）と、ツールとしての確実性（基本機能）を分離して定義する。

  • Core Layer: AIの有無に関わらず、動画・画像の変換ツールとして100%動作保証する。
  • AI Layer (PoC): Gemini Nano等の実験を行い、成功時のみAdaptive Strategyを有効化する。

2. 機能要件

(1) マルチメディア処理エンジン Core

v3.1で省略された詳細仕様を再定義。

  • WebCodecs API (Dual Stream): 復活
    • 映像(VideoEncoder)に加え、音声(AudioEncoder)の圧縮処理を実装。
    • MP4Box.js を使用し、処理後のストリームをMP4コンテナに結合(Muxing)する。
  • Image Optimizer: 復活
    • 動画サムネイルや資料画像のドラッグ＆ドロップを受け付け、WebP/JPEGに軽量化する機能。
  • MediaInfo Analysis: 復活
    • MediaInfo.jsを内蔵し、コーデック・プロファイル・色空間の詳細メタデータを表示する。

(2) システム基盤 & 安定性

  • PWA (Offline Capability): 復活
    • Service Workerを実装し、会場のWi-Fi切断時でもアプリの起動・処理を可能にする。
  • Memory Manager: 復活
    • 生成した動画Blobを明示的に破棄する「メモリ解放ボタン」を設置。
    • iOS(iPhone 7等)向けに、処理完了時に自動でメモリを掃除するオプションを用意。

(3) AI制御 & 自己修復 (PoC Layer)

実装条件: 事前実験にて動作確認済みであること

  • Adaptive Strategy:
    • Gemini Nano有効時: 安全なJSON生成（テンプレート適用）。
    • Cloud/WebLLM有効時: 生のFFmpegコマンド生成とSelf-Healing（エラー修正）の実演。
  • Fallback: AI全滅時は、ハードコードされた「推奨プリセット（松竹梅）」モードへ自動移行。

(4) エクスポート (The Souvenir)

  • Gist & Litterbox Integration: 復活
    • GitHub Gist: 生成されたコードやColab Notebookを保存（"Open in Colab"用）。
    • Litterbox/File.io: 一時的なファイル共有用。
    • これらをQRコード化してスマホで持ち帰り可能にする。
